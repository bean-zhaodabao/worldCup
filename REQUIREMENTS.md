## 一、系统架构

| 端 | 技术栈 | 运行形态 | 目录 |
|---|--------|---------|------|
| 用户端 | uni-app (Vue3) | App / 小程序 | `kaiyun/` |
| 管理端 | Vue3 + Element Plus | PC Web | `admin/` |
| 后端 | uniCloud (阿里云) | Serverless | `kaiyun/uniCloud-alipay/` |

## 二、管理端（PC端）功能

### 2.1 登录与账号管理
- 管理员登录
- **手动创建移动端用户账号**（用户名 + 密码），无需手机号绑定
- 管理员不分级，单一角色

### 2.2 赛事管理
- 字段：赛事名称、主队名称、客队名称、主队国旗图片、客队国旗图片、**开始时间**（下注截止时间）
- 赛事状态：**未开始 → 进行中 → 已结束 → 已结算**
  - 下注截止时间为赛事开始时间，比赛开始后不可下注
  - 不需要结束时间字段
- 管理员可将某赛事的所有玩法**复制**到另一场未开始的赛事，复制后可编辑

### 2.3 玩法分类管理（2级）
- **大类 → 小类** 两级结构
- 示例：
  ```
  大类：进球
    ├── 小类：总进球数区间  → 玩法：0-1/赔率3.60、2-3/赔率1.87、4+/赔率5.20
    ├── 小类：准确进球数    → 玩法：0球/赔率x、1球/赔率x、2球/赔率x...
    └── 小类：上半场准确进球数 → ...
  ```
- 玩法管理字段：关联赛事、关联小类、玩法名称、标识、赔率

### 2.4 中奖判定
- **管理员手动设定**每个玩法是否中奖（客户确认：管理员根据实际比赛结果设置）
- 一个赛事可以有多个玩法分别判定中奖
- 中奖判定**可修改**，修改后所有统计数据动态重算
- **修改需记录操作日志**

### 2.5 赔率管理
- 管理员可**动态调整赔率**
- 赔率修改后，**已有订单按旧赔率结算**（下单时快照赔率）

### 2.6 订单管理
- 按赛事 / 用户搜索订单
- 订单状态流转：**待开奖 → 已中奖 / 未中奖 → 已结算**
- 结算由管理员按赛事操作：设置某赛事"已结算"，该赛事所有订单自动变为"已结算"
- 统计：某用户当天下单总额、某用户中奖总额

### 2.7 报表统计
- 每日总流水
- 按赛事统计（下注总额、中奖总额、盈亏）
- 按玩法统计
- 盈亏报表
- 历史数据永久留存

### 2.8 系统配置
- 下注金额上下限（默认：2.00 ~ 100,000.00）
- 其他可配置项

### 2.9 操作日志
- 记录所有敏感操作：中奖设定、赔率修改、赛事状态变更、用户创建等

## 三、用户端（移动端）功能

### 3.1 登录
- 使用后台分配的用户名 + 密码登录

### 3.2 赛事及玩法查看
- 赛事列表 → 选择赛事 → 查看该赛事下的玩法
- **未设置玩法的赛事，用户端不可见**

### 3.3 下单
- 只能下注**未开始**的赛事（比赛开始后截止）
- 下单交互（底部弹出框）：
  - 玩法名称 + 赔率
  - 2级分类名称（大类 / 小类）
  - 对决双方名称
  - 金额输入框（单位 RMB）
  - 快捷金额标签
  - 带小数点的数字键盘
  - **滑动提交按钮**（防误触），按钮上显示：**可赢额度 = 下注额 × 赔率 - 下注额**
  - **串按钮**（Parlay / 串关）：选择多个玩法，赔率相乘，全部猜中才算赢

### 3.4 串关（Parlay）规则
- 同一订单可包含**多个不同玩法**
- 赔率为所选玩法赔率的乘积
- 必须全部猜中，该订单才算中奖
- 任一玩法未中奖，整单算未中奖

### 3.5 下注限制
- 单注金额范围由后台配置（默认 2.00 ~ 100,000.00）
- **不校验用户余额**（本项目不管理资金，只统计数据）

### 3.6 订单查询
- 查看个人订单列表：单号、下单时间、下注金额、玩法类型及赔率
- 只可查看自己的订单

### 3.7 不涉及的功能
- ❌ 充值 / 提现 / 支付
- ❌ 用户账户余额
- ❌ 余额校验

## 四、业务规则汇总

| # | 规则 | 说明 |
|---|------|------|
| 1 | 无资金体系 | 不下挂余额，不做支付，仅统计数据 |
| 2 | 无余额校验 | 下注不校验余额 |
| 3 | 中奖人工判定 | 管理员手动设定玩法是否中奖 |
| 4 | 一赛事多玩法 | 多个玩法各自独立判定中奖 |
| 5 | 玩法可复制 | 赛事玩法可复制到另一未开赛赛事 |
| 6 | 中奖可修改 | 修改后动态重算统计，记操作日志 |
| 7 | 串关（Parlay） | 一单多玩法，赔率相乘，全中方赢 |
| 8 | 下注截止=开赛时间 | 赛事开始后不可下注 |
| 9 | 三级赛事状态 | 未开始→进行中→已结束→已结算 |
| 10 | 赔率动态调整 | 已有订单按旧赔率结算（快照） |
| 11 | 管理员结算 | 按赛事维度触发结算 |
| 12 | 下注限额 | 后台可配，默认2~100000 |
| 13 | 赛事取消/延期 | 订单保持待开奖，不做处理 |
| 14 | 历史留存 | 所有数据永久保留 |
| 15 | 操作日志 | 记录敏感操作 |
| 16 | 无玩法赛事不可见 | 未设置玩法的赛事对用户隐藏 |
